## ⚠️ 已知问题 / 技术债务

### UI 刷新问题（未解决）
**问题描述：** 双击文件夹导航后，代码逻辑正确执行（日志确认），但 UI 不刷新，始终停留在原目录。

**尝试过的方案：**
1. `Files.Clear()` + `foreach Files.Add()` — 不生效
2. `Files = new ObservableCollection<FileItem>(items)` 替换引用 — 不生效
3. 加 `OnPropertyChanged(nameof(Files))` 强制通知 — 不生效
4. `Dispatcher.UIThread.InvokeAsync` 包裹所有 UI 更新 — 逻辑正确但 UI 仍不刷新

**分析：** 可能与 Avalonia 11 + Compiled Bindings (`x:DataType`) 的深层缓存机制有关。日志显示数据完全正确，PropertyChanged 通知也触发了，但 ListBox UI 不重绘。

**建议：**
- 换其他 UI 框架（Tauri、WPF、Qt）可能更合适
- 如果继续用 Avalonia，可尝试禁用 Compiled Bindings（移除 `x:DataType`）改用传统反射绑定

### 7. 新增仓库方式（网络协议）
**目前仅支持本地路径添加仓库**，需要新增网络协议方式：

| 输入项 | 说明 |
|--------|------|
| 仓库 URL | HTTP/HTTPS/SVN 协议地址 |
| 用户名 | SVN 用户名 |
| 密码 | SVN 密码 |
| 仓库名称 | 用于生成本地目录名 |
| 本地路径 | 由程序自动推导：`~/.svnfilemanager/workcopies/{仓库名称}/`（用户无需指定）|

**凭证存储：**
- 用户名明文存储于 `repositories.json`
- 密码**加密存储**（使用 DPAPI / Windows Credential Manager 或类似安全存储）
- 配置保存路径：`~/.svnfilemanager/repositories.json`

**实现方式：**
1. 弹窗让用户填写仓库名称 + URL + 凭证
2. 调用 `svn checkout --username <user> <url> <local-path>`
3. checkout 完成后将新路径加入仓库列表

**交互方案：**
- ✅ 两个独立按钮："添加本地仓库" + "从网络检出"
- ✅ 左侧栏头部按钮列表（从上到下）：
  1. 添加本地仓库
  2. 从网络检出
  3. ────────────
  4. 配置
  5. 关于
